# Methodology

This skill keeps methodology implicit in the workflow. Use these methods only to guide questioning; do not force the user to learn or name them.

## Framework Semantics

Use to uncover the scenario, roles, objects, actions, states, and relationships behind a requirement.

## Use Case Modeling

Use to identify actors, triggers, normal paths, branch paths, and exception paths.

## DDD / Event Storming

Use to identify core objects, state transitions, domain events, business rules, dependencies, and conflicts.

## Quality Attribute Scenarios

Use to check key non-functional constraints such as performance, security, compatibility, observability, maintainability, rollback, and recovery.

## Service Blueprint

Use to understand downstream impacts on users, R&D, testing, marketing, and service.

## Working Principle

Prioritize the few questions that most affect downstream understanding. Do not pursue completeness for its own sake.
