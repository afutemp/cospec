# 用户旅程设计 — 中文参考说明

> 本文件仅供人阅读，不加入 AI 上下文。AI 使用的是同目录下的 `SKILL.md`（中文）。

## 概述

把已通过评审、决定要做的用户需求，细化为可用于需求文档、方案设计和用户需求评审的用户旅程设计产物。

## 核心机制

### 强制分阶段确认

Skill 必须保持状态机工作，每轮只能处于一个阶段：

1. **Step 0**：输入理解确认
2. **Step 1**：方案设计共创
3. **Step 2**：未来旅程共创
4. **Step 3**：最终文档生成

没有用户明确确认上一阶段，不得进入下一阶段。

### 关键约束

- 有输入物不等于已确认
- 不确定信息必须显式标注为"待验证假设"或"待确认事项"
- 方案必须覆盖老客户升级影响（兼容性、灰度、回滚、配置继承）
- 必须逐竞品对比，不能泛写

## 输出物

需求背景、方案设计、未来旅程（主流程/分支流程/异常流程）、目标达成分析。
