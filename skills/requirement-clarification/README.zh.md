# 需求澄清 — 中文参考说明

> 本文件仅供人阅读，不加入 AI 上下文。AI 使用的是同目录下的 `SKILL.md`（英文）。

## 概述

在产品经理写用户需求文档之前，帮助把初始需求想法转化为"想全面"的澄清结果。通过聚焦多轮提问，让用户、研发、测试、营销和服务等各方影响显式化。

## 工作流

1. 填写需求输入模板
2. 首轮诊断：识别事实、假设、缺口，提出不超过 5 个关键问题
3. 聚焦追问：每轮不超过 5 个问题，只追问影响用户/研发/测试/营销/服务理解的内容
4. 范围控制：澄清阶段不展开为完整需求文档
5. 收敛输出：信息充分时，输出需求背景澄清交付物

## 输入要求

- 需求名称、一句话需求、产品线/模块
- 目标用户、使用场景、当前问题、期望结果
- 已知约束、前序材料

## 输出物

需求概述、全景结论、异常边界与风险、下游影响、事实/假设/待确认、3-5 条核心共识。
