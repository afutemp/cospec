## S1 问题本质讨论（对话中完成，不写入输出）

Agent 必须先与用户讨论：
1. 拆解问题（架构问题 / 策略问题 / 交互问题）
2. 追问底层目标（表面问题 ≠ 真正要决策的点）
3. 界定边界（哪些是业界共识？哪些是厂商定位决定的？）
4. 反对预设方案（用户带着方案来问时提醒回归问题本质）

## S4 验证（对话中完成，不写入输出）

输出完成后必须询问：以上是否解答了原始问题？是否需要补充？
