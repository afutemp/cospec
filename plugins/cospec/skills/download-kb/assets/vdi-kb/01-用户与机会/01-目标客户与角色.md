# 目标客户与角色

## 一、目标客户

`【推导·待审】` 基于 VDC 控制台功能定位，目标客户可能包括：

| 客户类型 | 典型场景 | 来源 |
| :--- | :--- | :--- |
| 中大型企业 IT 部门 | 需要统一交付办公桌面、远程应用和文件服务 | `【推导·待审】` |
| 教育机构 | 课程管理、计算机教室、虚拟实验室（行业应用未启用） | `【事实·界面】` |
| 设计与制造行业 | 需要 3D 工作站和 GPU 资源的托管物理机 | `【事实·界面】` |
| 对安全合规要求高的组织 | 需要外发审计、接入安全、策略控制、分布式防火墙 | `【事实·界面】` |

## 二、管理员角色

当前梳理基于 `admin / Super Admin` 角色。其他角色待验证。

| 角色 | 可见/操作范围（当前已知） | 来源 |
| :--- | :--- | :--- |
| Super Admin | 资产、服务、策略、安全增强、运维中心全模块可见 | `【事实·界面】` |
| 普通管理员 | 菜单和操作权限差异未验证 | `【待验证】` |
| 受限管理员 | 菜单和操作权限差异未验证 | `【待验证】` |
| 审计管理员 | 可下载审计文件导出工具、查看报表中心审计数据 | `【事实·界面】` |

## 三、终端用户/资源使用角色

| 角色 | 说明 | 来源 |
| :--- | :--- | :--- |
| 本地用户 | VDC 本地创建的账号，可分配角色和资源 | `【事实·界面】` |
| 域用户 | 通过 AD/LDAP 导入的域账号 | `【事实·界面】` |
| 安全组 | AD 安全组或本地用户组，可批量关联策略和资源 | `【事实·界面】` |
| RADIUS 用户 | 通过 RADIUS 外部认证接入的用户 | `【事实·界面】` |
