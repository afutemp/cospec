# Product Planning KB Semantic Rubric v1.0.0

Total semantic weight: 55.

| Dimension | Max |
|---|---:|
| Strategy and value clarity | 8 |
| User/problem/evidence quality | 10 |
| Feature problem-goal-scenario closure | 12 |
| Release scope and roadmap consistency | 10 |
| Quality/dependency/risk/validation closure | 8 |
| No fabrication and transparent gaps | 7 |

Each dimension returns `id`, `score`, `maxScore`, `reason`, `evidence[]`, and `findings[]`. Evidence includes a local document path and source IDs, or explicitly states source insufficiency. Evaluators actively seek contradictions, Tech promoted to customer value, comments overriding issue text, and claims absent from the snapshot.

When only selected dimensions are evaluated, applicable semantic points are normalized to the fixed 55-point semantic layer. Reports record selected dimensions so partial coverage remains visible.
