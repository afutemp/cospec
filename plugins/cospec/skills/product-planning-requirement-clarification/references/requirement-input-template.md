# Requirement Input Template

Use this template when product planning needs to provide the minimum input for the "想全面" stage.

```text
【需求名称】
例如：支持生命周期策略可视化配置

【一句话需求】
希望系统支持什么能力？解决什么问题？

【产品线/模块】
例如：对象存储 / 云平台 / 安全运营 / 运维管理 / 不确定

【目标用户】
谁会使用？例如管理员、租户、运维人员、业务用户、服务人员等。

【使用场景】
用户在什么场景下会用到这个能力？

【当前问题】
现在用户怎么做？痛点是什么？

【期望结果】
这个需求做好后，用户应该得到什么结果？

【已知约束】
有哪些边界、限制、版本要求、兼容要求或暂时不做的内容？

【前序材料】
可选：立项材料、客户反馈、竞品分析、用户旅程、共创结论等。
```
