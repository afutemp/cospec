# 产品知识库查询示例

本文件展示 `product-kb-query` SKILL 的典型查询场景和预期输出格式。

## 示例 1：功能类查询

**查询：** "认证方式有哪些？"

**检索路径：**
1. 根据问题类型映射到 `03-功能规划/`。
2. 在该目录下查找文件名包含“服务”或“认证”的文档，例如 `02-服务功能.md`。
3. 读取文档中关于认证模块的章节。

**预期回答格式：**

根据 `03-功能规划/02-服务功能.md`，认证模块包含 14 个认证子页：

1. 密码安全
2. 外部 LDAP/RADIUS
3. 证书 / USB Key
4. 客户端域单点登录
5. 深信服 aTrust
6. 外部认证
7. 短信认证
8. 动态令牌
9. 硬件 ID
10. 本地密码
11. TOTP 认证
12. 统一认证
13. 联邦认证
14. 第三方连接

其中主认证可设置为“全部满足”或“任一满足”；次认证包括硬件 ID、短信认证、动态令牌、TOTP。

> 来源：`03-功能规划/02-服务功能.md`

## 示例 2：风险类查询

**查询：** "知识库里有没有提到风险？"

**检索路径：**
1. 根据问题类型映射到 `05-协作与依赖/`。
2. 读取 `01-依赖风险与协作.md` 中的风险清单部分。

**预期回答格式：**

根据 `05-协作与依赖/01-依赖风险与协作.md`，主要风险包括：

| 风险 ID | 风险 | 类别 | 风险等级 | 预防措施 |
| :--- | :--- | :--- | :--- | :--- |
| RK-001 | 模板更新导致派生虚拟机系统盘数据丢失 | 技术/数据 | 高 | 更新前提示影响范围；提供仅更新模板选项 |
| RK-002 | 接入安全/情景策略配置错误导致合法用户被阻断 | 可用性 | 高 | 提供策略影响范围预览；支持禁用/回滚 |
| RK-003 | 分布式防火墙启用影响业务网络访问 | 可用性 | 高 | 启用前二次确认；支持直通模式 |

## 示例 3：范围总结类查询

**查询：** "总结一下项目范围"

**检索路径：**
1. 宽泛总结类问题优先读取根目录 `README.md`。
2. 再读取 `02-规划与范围/02-版本范围与路线图.md`。

**预期回答格式：**

根据 `README.md` 和 `02-规划与范围/02-版本范围与路线图.md`：

- **已覆盖模块**：资产、服务、策略、安全增强、运维中心。
- **验证深度**：页面已读取、交互已展开，未执行保存/删除/启停等变更操作。
- **明确不在当前范围内**：分布式防火墙规则表单、行业应用课程管理、文件库存储与文件操作、软件分发应用容器测试步骤、普通/受限管理员视角。

## 示例 4：不确定信息处理

**查询：** "普通管理员能看到哪些菜单？"

**检索路径：**
1. 根据问题类型映射到 `01-用户与机会/01-目标客户与角色.md`。
2. 发现文档中权限矩阵对应单元格为“待确认”。

**预期回答格式：**

根据 `01-用户与机会/01-目标客户与角色.md`，普通管理员、受限管理员、只读管理员的具体权限边界标注为 `[OPEN]` / 待确认。

> 当前知识库仅基于 Super Admin 视角梳理，未使用普通管理员/受限管理员账号验证菜单及操作权限差异。如需准确答案，需补充对应角色的权限验证。

## 示例 5：被其他 skill 调用时的输入

当 `product-planning-requirement-clarification` 在执行前调用本 skill 时，可能收到如下查询：

**查询：** "请总结与当前需求相关的产品战略、目标客户、已有功能、已知风险和约束。"

**预期回答格式：**

根据 `00-综述/01-产品战略与价值.md`、`01-用户与机会/01-目标客户与角色.md`、`05-协作与依赖/01-依赖风险与协作.md`：

- **产品战略**：本产品定位为……
- **目标客户**：主要面向……
- **已有功能**：资产、服务、策略、安全增强、运维中心五大模块。
- **已知风险**：模板更新导致派生虚拟机系统盘数据丢失（RK-001，高）、接入安全/情景策略配置错误导致合法用户被阻断（RK-002，高）等。
- **约束**：当前权限矩阵为 `[OPEN]`，需后续补充。

> 来源：`00-综述/01-产品战略与价值.md`、`01-用户与机会/01-目标客户与角色.md`、`05-协作与依赖/01-依赖风险与协作.md`
